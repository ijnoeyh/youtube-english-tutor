## 1. Listen "다시 듣기"가 문장 끝에서 안 멈추던 버그

**증상**: 구간 재생 시 시작 위치로는 잘 이동하는데 `end_sec`에서 멈추지 않음.

**진짜 원인 (핵심)**: 반응형 레이아웃이 데스크탑용·모바일용 두 벌 모두 마운트돼서
`<YouTube>` 플레이어 인스턴스가 **2개** 생성됨. `onReady`가 두 번 발생해 `setPlayer`가
경쟁(race)하면서, 제어 대상 `player`가 *화면에 안 보이는* iframe을 가리킬 수 있었음.
→ 사용자가 보는 영상과 `getCurrentTime()`을 읽는 플레이어가 달라 정지 트리거가 안 걸림.

**어떻게 고쳤나**:
- `app/page.tsx`: `useIsDesktop()` 훅(`matchMedia`) 추가 → 화면 폭에 맞는 레이아웃 **하나만**
  렌더 → 플레이어 항상 1개 보장.
- `components/listen/ListenMode.tsx`: 정지 시각을 `stopAtRef`(ref)에 담아 폴링 콜백이 항상
  최신값을 읽게 하고, `getCurrentTime` 과도기값(NaN) 가드 + 폴링 100ms로 정확도 향상.

**배운 개념**: Tailwind `hidden`/`lg:hidden`은 *시각적으로만* 숨길 뿐 컴포넌트를 언마운트하지
않는다. 부수효과(iframe·타이머)를 가진 요소를 양쪽 레이아웃에 두면 인스턴스가 중복 생성된다.

## 3. 자막을 "문장 단위"로 병합

**문제**: YouTube 자막은 문장이 아니라 *화면 타이밍* 단위로 잘려 있어
("I think the most" / "important thing is" / ...) 학습 시 문장이 어색하게 끊김.

**어떻게**: `backend/utils/youtube.py`에 `merge_into_sentences()` 추가.
- 누적 버퍼(accumulator)로 자막 큐를 모으다가 아래 중 하나면 한 문장으로 확정(flush):
  1. 문장 종결부호(`.` `!` `?`, 뒤 따옴표 포함)로 끝남
  2. 길이 안전장치(`max_chars`) 초과 — *자동생성 자막은 구두점이 없어서 필요*
  3. 큐 사이 시간 간격(`max_gap`) 초과 — 장면 전환
- 타이밍 재계산: 병합 문장의 `start`=첫 큐, `end`=마지막 큐 → "다시 듣기"가 문장 전체 재생.
- `backend/routers/videos.py`의 `register_video`에서 `fetch_transcript → merge_into_sentences
  → DB 저장` 순으로 연결.
- 튜닝: `max_chars=140`, `max_gap=1.2` (shorts 호흡에 맞춤).

**⚠️ 주의**: 병합은 *새로 등록하는 영상*부터 적용. 기존 영상은 삭제 후 재등록해야 깔끔해짐.

**배운 개념**: 자막은 수동(구두점 O)/자동생성(구두점 X) 두 종류라, 구두점만으로 나누면
자동 자막에서 무한 누적된다 → 길이/간격 안전장치를 함께 둬야 한다.

**배운 개념**: `datetime.utcnow()`로 저장된 UTC 시각을 "연속 학습일"처럼 사용자 체감

## 6. 폰·PC HTTPS 접속 배포 (권한 없는 컨테이너)

**제약**: 작업 환경이 *권한 없는 Docker 컨테이너 안*이라 Docker 실행 불가
(`/var/run/docker.sock` 없음, privileged 아님). 또 Speak의 마이크(`getUserMedia`)와 TTS는
**HTTPS(보안 컨텍스트)**에서만 동작.

**해결 구조** (Docker 없이 프로세스로 직접 실행 + Tailscale 터널):
```
폰/PC (Tailscale 로그인)
  │ HTTPS (유효 인증서 자동, *.ts.net)
  ▼
tailscale serve (userspace 모드 — 권한 불필요)  →  127.0.0.1:3001
  ▼
next start :3001  ── /api/* ─(Next rewrites)→ uvicorn :8001
  └ 그 외 → 프론트
```
- 배포본은 dev 서버(프론트 3000/백엔드 8000)와 **포트 분리**(3001/8001)해 공존.
- `next.config.mjs`에 `rewrites`(/api→백엔드) + `distDir`(빌드 분리) + `eslint.ignoreDuringBuilds` 추가.
- 접속 주소: `https://english-tutor.tail656ed.ts.net` (타넷 전용, 외부 비공개).
- 재기동: `./run-app.sh` (컨테이너 재부팅 시).

**핵심 함정 2가지 (디버깅하며 발견)**:
1. **Next.js rewrites는 빌드 타임에 굳는다** (`routes-manifest.json`). 백엔드 주소를 바꾸려면
   `next start` 환경변수가 아니라 *빌드*에 `BACKEND_INTERNAL_URL`을 넣어야 한다.
2. **uvicorn `--reload`가 꼬이면** 새 워커가 떠도 원본 워커가 소켓을 계속 쥐어 옛 코드가
   서빙된다. `openapi.json`의 실제 라우트 목록으로 진단했고, 배포본은 reload 없는 8001로 분리.

**배운 개념**: 호스트 권한이 없어도 *아웃바운드 터널*(Tailscale userspace)은 동작한다 —
이게 권한 없는 컨테이너에서 폰 HTTPS 접속을 가능케 한 핵심.

```bash
# 배포본 재기동 (컨테이너 재부팅 후 등)

# 상태 확인
ss -ltn | grep -E ':(3001|8001)'                       # 배포 프론트/백엔드
tailscale --socket=/var/run/tailscale/tailscaled.sock serve status

# 로그
tail -f /tmp/fe-prod.log /tmp/backend-prod.log

# dev 백엔드 "Address already in use" 시: 8000 점유 프로세스 종료 후 재기동

```
